Two-Lane Launch Option

A launch structure now under active exploration: keep ordinary names simple, while deferring salient existing names into a separate auction lane where the market discovers required BTC commitment under fixed long lock rules.

Core Idea

Split launch into two lanes:

Ordinary Lane

  • Names not on a reserved list
  • Simple commit / reveal
  • Objective floor table
  • Standardized lock period

Reserved Lane

  • Broad deferred list of salient names
  • Unavailable on launch day
  • Unlock later at pre-announced heights
  • BTC amount discovered by auction

The key shift is that the reserved list decides which names are special and when they unlock, not what each one should cost.

Why Explore This

The main fairness objection is not whether someone can corner the whole namespace. It is whether obvious existing names can be acquired too cheaply.

This model treats speculative competition as part of the solution: speculators can bid away easy upside even if the natural operator is absent at first, as long as the winning commitment carries real capital-time cost.

The system does not need to guarantee the real operator wins day one. It needs to avoid obvious cheap giveaways and underpriced hostage opportunities.

Ordinary Lane

  • Preserves DNS-like “I saw it and can try to get it now” UX.
  • Keeps long-tail and personal names tractable under a simpler standardized ordinary lock, such as 1 year.

Reserved Lane

  • Reserved list announced before launch.
  • Names withheld from ordinary commit / reveal.
  • Auctions begin later at known block heights.
  • Protocol fixes duration; auction discovers BTC amount.
  • Reserved minimum = max(ordinary floor, reserved-class floor).

This lets the reserved list be broader than a hand-priced premium table, because inclusion no longer implies “we custom-priced this name.”

Auction Dynamics

  • Reserved names only.
  • Likely fixed long lock, e.g. 10 years.
  • Bids compete on BTC amount only, not arbitrary BTC-time pairs.
  • First valid bid starts that name’s auction clock.
  • Long base window, likely weeks rather than hours, with soft close so late bids extend the clock.
  • One active bid per bidder per auction.
  • Your own earlier bid can be rolled upward by spending the old bid output and adding only the difference.

Why Soft Close

A hard final block invites end-sniping and miner-timing games. A soft close is closer to a live auction: if a higher bid arrives late, the auction extends and others get time to answer.

The protection is not forcing auctions to stop at all costs. It is making continued extension economically meaningful.

Open Questions

  • How broad should the reserved list be, and should it unlock in waves?
  • What fixed long duration should the reserved lane use?
  • What minimum increment rule prevents cheap endless extensions while keeping auctions healthy?
